# Prompt
card back graphic is not visible.  only a blueish color is shown

# Implementation
Recognized that the "blueish color" is actually the cached Material Blue (`#2196F3`) `<shape>` drawable I originally created for the card back in our very first implementation step. Because I completely changed the file type from a `<shape>` to a complex `<vector>`, Android Studio's 'Hot Swap' / 'Apply Changes' feature failed to push the new resource to the device. Advised the user to perform a Clean Project and Rebuild to flush the Gradle resource cache.
