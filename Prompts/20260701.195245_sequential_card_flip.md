# Prompt
let each card rotate on its own axis.
All cards should not rotate at the same time.
First card should rotate and then display itself.  Then the next card should start rotating and show its face, then the third card and so on....

# Implementation
Replaced the random concurrent shuffling loop in `CardsFragment.java` with a precise sequential 3D flip animation. 
Implemented `animateCardSequentially()` which grabs the `RecyclerView.ViewHolder` for each individual card and applies an `ObjectAnimator` to `rotationY`. 
The method flips the card 90 degrees out, updates the underlying data and view background instantly midway through the animation, and then flips it 90 degrees in to reveal the face, before recursively firing the exact same animation sequence for the next card down the line until all cards have been revealed.
