# Implementation
Added a visual "motion blur" effect to the intermediate random words that flash by during the pre-selection generation phase.

- **WordAdapter.java**: Injected an `isBlurring` flag. If true, the `TextView` dynamically applies Android 12+'s native hardware-accelerated `RenderEffect.createBlurEffect()` for a true gaussian blur, and simultaneously drops the alpha (opacity) to `0.7`. For older Android versions (API 24-30), it falls back to a heavy alpha transparency (`0.3f`) which simulates motion blur during rapid shuffling.
- **WordsFragment.java**: Wrapped the core `generateWords()` animation runnable with commands that set `adapter.setBlurring(true)` the moment shuffling begins, and revert to `false` (locking in sharp, 100% opaque text) the millisecond the final word is chosen.
