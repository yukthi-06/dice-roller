# Prompt
can we have a better animation for dice?

# Implementation
Re-engineered the dice rolling animation loop in `DiceFragment.java` to simulate physical tumbling.
Instead of a simple static randomization loop that relied on `notifyDataSetChanged()` (which destroys view animation states), I decoupled the visual shuffling from the adapter state:
- Each individual dice view is grabbed directly from the `RecyclerView` layout.
- The `ImageView` receives a concurrent `ViewPropertyAnimator` that rapidly scales up the dice (`1.3f`) and wildly rotates it (`360 * 4` degrees in a random direction) using an `AccelerateDecelerateInterpolator`.
- Midway, it seamlessly chains to a second animation dropping the dice back to original scale using a `BounceInterpolator`.
- Simultaneously, a recursive background handler manually swaps the `src` drawable of the spinning `ImageViews` every `80ms` to create a realistic blurring faces effect, entirely bypassing the adapter until the final result is locked in.
