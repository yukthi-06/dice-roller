# Prompt
when less number of cards or numbers are selected, do not stretch it wide.  Maintain origial size/width and align them to center

# Implementation
Re-architected the layout components for `item_dice.xml`, `item_card.xml`, and `item_number.xml`. 
Wrapped the core visual element (ImageView/TextView) inside a `FrameLayout` with `layout_width="match_parent"`. 
Locked the visual element's width to fixed dimensions (`100dp` or `90dp x 130dp`) and applied `android:layout_gravity="center"`. 
This allows the `GridLayoutManager` to stretch the invisible container perfectly into our requested pyramid structure without warping the actual graphic!
