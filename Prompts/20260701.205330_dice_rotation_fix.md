# Prompt
rotation is fine, but by the time the rotation slows down, the face value has to be fixed.  it should slow and stop with the selected number on its face

# Implementation
Decoupled the physical rotation duration from the scale bouncing duration using discrete `ObjectAnimator`s. 
Set the 5-spin `rotation` to last the *entire* animation duration using a `DecelerateInterpolator` so it gradually slows to a halt. 
Modified the background face-shuffling `Runnable` to terminate 400ms *before* the physical rotation completely stops. It instantly locks the dice `ImageView` to its final predefined face, allowing the user to clearly see the final number gracefully spinning down to its resting position.
