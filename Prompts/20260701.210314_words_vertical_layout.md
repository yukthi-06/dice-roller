# Prompt
the words should be in vertical .  dont try to fit horizontally

# Implementation
Reconfigured the Words layout to display items in a standard vertical scrolling list rather than the centered grid layout used by Alphabet and Numbers. 

- Replaced `GridLayoutManager` with a standard `LinearLayoutManager` in both `fragment_words.xml` and `WordsFragment.java`.
- Redesigned `item_word.xml` to stretch across the width of the screen (`match_parent` with margins) instead of being constrained to a small square.
- Removed auto-resizing text attributes since words now have ample horizontal space on their own line, opting for a larger fixed `24sp` text size to dramatically improve readability.
